Atividades da Garantia da Qualidade

Uma das formas de melhorar a qualidade do processo e do produto no final é investir nas atividades da garantia da qualidade de software. São elas:

Validação, Verificação e Teste de Software (V,V&T)

As atividades de validação, verificação e teste de software são realizadas desde o início do processo de software durante a coleta de requisitos. Afinal, problemas detectados logo no começo geram menor retrabalho no final e menos desperdício de recursos.

A Validação consiste em garantir que foi compreendido corretamente as necessidades do cliente para construir o produto que ele necessita. Neste caso estamos trabalhando com “o que”.

Podemos validar os requisitos com o cliente fazendo por exemplo um desenho da interface do sistema (protótipo) para que ele valide. Validar o produto ou parte pronta com o cliente e internamente por testes de software. Entre outras formas que veremos.

Enquanto a validação trabalha com “o que”, a Verificação deve garantir o “como”. Isto que dizer que durante a verificação estamos preocupados em averiguar se as soluções tecnológicas adotadas para solucionar o problema são as melhores considerando o custo e benefício para o projeto.

Já os Testes de Software consistem em validar se o comportamento realizado pelo sistema, ou parte dele (código-fonte), é o esperado perante as especificações dos requisitos. Neste caso examinamos o comportamento durante sua execução em relação as definições dos requisitos.

Mas como realizar essas atividades? Utilizamos o que chamamos de técnicas de revisão para realizar essas atividades.

As revisões de software são como um filtro para a gestão da qualidade, sendo aplicadas em várias etapas durante o processo de software, para eliminar os defeitos. Afinal, os defeitos impactam diretamente nos custos de um projeto.

Parada obrigatória

Para entender mais, leia a seção 20.1 Impacto de defeitos de software nos custos, do cap. 20 - Técnicas de revisão, do livro Engenharia de software: uma abordagem profissional, de Roger S. Pressman e Bruce R. Maxim. Bom estudo!

Você percebeu como são importantes as atividades que averiguam a qualidade durante todo o processo de desenvolvimento e como elas contribuem para a redução de consumo de recursos em um projeto de software?

Uma das ferramentas utilizadas para fazer a revisão formal de software é a inspeção por intermédio de checklist de software. O checklist é um conjunto de questionamentos para um determinado artefato de software e suas características que deve ser utilizado para guiar uma revisão formal. O resultado da revisão formal é um relatório que aponta os problemas encontrados no artefato e dúvidas que devem ser sanadas.

Parada obrigatória

Vamos entender melhor o que são técnicas de revisões formais e outras formas de realizar além do checklist. Para isso, leia a seção 20.6 Revisões técnicas formais, do cap. 20 - Técnicas de revisão, do livro Engenharia de software: uma abordagem profissional, de Roger S. Pressman e Bruce R. Maxim. Bom estudo!
